# 通信协议整理

> 说明：
> - 当前项目的 ROS 串口链路以“当前固件 ROS UART 协议”为维护基准。
> - 后面的厂家协议表格是历史资料转写，保留用于对照，不作为当前 ROS 链路的最终依据。
> - 个别表头中的方向字段存在 `C63A` / `C50A` 不一致的情况，这里按原图保留，不额外修正。
> - 为便于 Codex 识别，采用“协议名 + 基本信息 + 字节定义”的统一格式。
> - 当前固件只在 `UART4` ROS 链路使用协议 1 和协议 4：上行固定 24 字节基础帧，下行固定 11 字节控制帧。
> - ICM20948 已移除；上行帧中的原 IMU 字节仍保留为协议占位，但当前固定为 `0`。
> - 协议 2、协议 3 和协议 5 仅作为厂家历史资料保留；当前固件已移除 `Ranger` 和 `AutoRecharge`，不会在 ROS 串口混发 19 字节或 8 字节附加帧。
> - Hall 调试数据不再通过 UART4 追加发送；调试请看 OLED 或 Keil Watch 的 `g_hall_speed_state`。

## 当前固件 ROS UART 协议

### STM32 -> ROS 上行基础帧

- **文件依据**：`WHEELTEC_APP/data_task.c`
- **方向**：STM32 -> ROS
- **串口**：`UART4`
- **总长度**：固定 24 字节
- **帧格式**：

```text
0x7B flag vx_h vx_l vy_h vy_l wz_h wz_l imu0 ... imu11 bat_h bat_l bcc 0x7D
```

| 字节 | 当前固件含义 | 数据来源 | 上位机处理预期 |
|---|---|---|---|
| 0 | 帧头 `0x7B` | 固定值 | 会解析，用于同步帧头 |
| 1 | 控制/反馈有效标志，当前固定为 `1` | 固定值 | 会解析，但不是实时使能状态 |
| 2..3 | `vx = raw / 1000.0`，单位 `m/s` | 真实霍尔轮速；无有效霍尔测速时填 `0` | 会解析为线速度 X |
| 4..5 | `vy = raw / 1000.0`，单位 `m/s` | 固定 `0`，当前阿克曼底盘没有横向速度 | 会解析，但当前没有真实数据 |
| 6..7 | `wz = raw / 1000.0`，单位 `rad/s` | 由霍尔 `vx`、当前舵机 PWM 和轴距估算；不是 IMU 实测 | 会解析为 Z 角速度 |
| 8..19 | 原 IMU 字段占位 | 固定 `0`，ICM20948 已移除 | 旧上位机会按 IMU 字段读到零值 |
| 20..21 | 电池电压，单位 `mV` | ADC 换算：`ADC / 4095 * 3.3 * 11` | 会解析为电池电压 |
| 22 | 前 22 字节 BCC 异或校验 | 运行时计算 | 会校验 |
| 23 | 帧尾 `0x7D` | 固定值 | 会解析，用于同步帧尾 |

当前上行速度反馈的边界：

- `vx` 是唯一有直接传感器支撑的运动速度字段，来自 `HallSpeed_GetSignedSpeedMps()`。
- `wz` 是运动学估算值，来自 `telemetry_estimate_vz_from_pwm(feedback_vx, g_state.servo_pulse_us)`。
- `vy` 和 IMU 字段只是为了保持 24 字节协议兼容，不代表真实测量。
- 不允许在 `UART4` ROS 数据流里追加调试帧，否则旧上位机固定长度解析会错位。

### ROS -> STM32 下行控制帧

- **文件依据**：`WHEELTEC_APP/SerialControl_task.c`
- **方向**：ROS -> STM32
- **串口**：`UART4`
- **总长度**：固定 11 字节
- **帧格式**：

```text
0x7B cmd1 cmd2 vx_h vx_l vy_h vy_l wz_h wz_l bcc 0x7D
```

| 字节 | 当前固件含义 | 控制效果 |
|---|---|---|
| 0 | 帧头 `0x7B` | 必须匹配，否则不会进入 11 字节帧解析 |
| 1 | `cmd1` | `0x00` 才作为速度控制；`0x04` 作为旧 RGB 帧识别后忽略；其他值忽略 |
| 2 | `cmd2` | bit7 作为 `flag_stop`，会影响停车；bit0 当前解析后忽略 |
| 3..4 | `vx_mps = raw / 1000.0` | 有效，影响电调 PWM |
| 5..6 | `vy_mps = raw / 1000.0` | 会解析并参与“零速度”判断，但阿克曼输出不使用横向速度 |
| 7..8 | `vz_rad_s = raw / 1000.0`，即底盘 Z 角速度 | 有效，影响舵机转角映射 |
| 9 | 前 9 字节 BCC 异或校验 | 必须匹配，否则整帧丢弃 |
| 10 | 帧尾 `0x7D` | 必须匹配，否则整帧丢弃 |

下行解析规则：

- 只有 `cmd1 = 0x00` 会更新当前 ROS/Orin 速度命令。
- `cmd1 = 0x04` 的旧 RGB 控制帧仍会被识别并忽略，避免旧上位机命令破坏 11 字节解析；当前固件不再启动 RGB APP 任务。
- `cmd1` 为其他值时，帧通过长度、帧尾和 BCC 检查后仍会被丢弃。
- 任意 `UART4` 字节流中还会扫描文本命令 `reset` 和 `LOG0`..`LOG3`，它们不是 ROS 二进制帧的一部分。
- RC 接管激活时，非零 ROS 运动命令会被阻断；零速度或 `flag_stop` 仍允许进入，用于释放/停车。
